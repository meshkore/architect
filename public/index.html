A

Run the Python daemon (recommended)

canonical · no install

A single stdlib-only script. Works on macOS, Linux, and Windows. No pip, no npm, no installer — so corporate machines that block binaries are fine. Any agent in your editor (Claude Code, Cursor, Windsurf, …) can run it inside the repo.

From the root of a repo that already has a .meshkore/ folder, run:

# one-time: download into .meshkore/scripts/
mkdir -p .meshkore/scripts
curl -fsSL https://meshkore.com/reference/cluster/scripts/daemon.py -o .meshkore/scripts/daemon.py

# every time: start it
python3 .meshkore/scripts/daemon.py

It binds the first free port in 5570–5589 (so multiple projects coexist), mints a bearer token at .meshkore/credentials/portal-token, and rebuilds state.json from your markdown on the fly. Spec §10.2.

First time on this repo

No .meshkore/ folder yet? Open your AI coding tool at the repo and paste this prompt. It scaffolds the standard layout, downloads the Python daemon, and starts it.

Apply the MeshKore Standard to this repo. Read https://meshkore.com/standard (canonical) and follow it: (1) create the .meshkore/ tree per §2, including .meshkore/public/cluster.yaml from the dev template at https://meshkore.com/reference/cluster/templates/cluster.yaml.dev, (2) drop the editor boot block from §8 at the repo root, (3) download the Python daemon from https://meshkore.com/reference/cluster/scripts/daemon.py into .meshkore/scripts/daemon.py and start it with `python3 .meshkore/scripts/daemon.py`, (4) print the bearer token from .meshkore/credentials/portal-token.

Advanced: legacy Node binary (meshcore)

The original meshcore binary still works and adds capabilities the Python daemon doesn't have yet (agent dispatch, AI-runner pool, full chat). Download from download.meshkore.com and run meshcore start. Some corporate machines block unsigned binaries — if yours does, fall back to Path A.
